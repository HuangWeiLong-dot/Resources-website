# 已知问题与缺陷记录（Resources Hub）

- **对应版本**：`v0.1.0`

> 记录当前版本中已发现但尚未解决，或曾出现并已修复的 bug，方便后续维护。

## 颜色与主题相关

- **`color-mix` 兼容性**
  - 描述：部分旧版浏览器（例如早期 Edge）可能不完全支持 `color-mix(in srgb, ...)`，会导致卡片边框、标签背景或光线效果颜色与主色不一致。
  - 现状：当前项目主要面向现代浏览器，暂未提供退化方案。
  - 建议：如需兼容，可使用 `@supports (color: color-mix(...))` 包裹相关样式，并为不支持的场景提供固定备用颜色。

- **主题滑杆样式重复**
  - 描述：`index.html` 与 `collection.html` 中内联 `<style>` 区块均包含一套 `theme-panel` 样式，存在重复。
  - 风险：若日后修改，只改动其中一处会导致两页视觉不一致。
  - 计划：后续应将这部分样式抽离到 `styles/main.css` 中统一维护，并删除页面内联样式。

## 动画与交互

- **前进 / 后退导致重复入场动画**
  - 描述：页面使用 `stagger-item` + `fade-in-stagger` 做入场动画，当浏览器通过「前进 / 后退」返回页面时，部分元素会重新播放入场动画，与用户心理预期不完全一致。
  - 现状：整体影响较小，暂未处理。
  - 建议：可以在 `sessionStorage` 记录「是否已播放入场动画」，二次进入时跳过动画或缩短时长。

- **Hero 光线效果性能**
  - 描述：`collection-hero` 与资源卡片在 `mousemove` 上实时更新 CSS 变量并触发渐变重绘，在低性能设备或开启大量标签页时可能出现轻微掉帧。
  - 建议：必要时可以：  
    - 通过 `requestAnimationFrame` + 节流减少更新频率；  
    - 在移动端或窄视口直接禁用该效果（通过媒体查询或 UA 识别）。

## 已修复问题（历史记录）

- **资源卡片光线离开时闪回中心**
  - 现象：鼠标在卡片边缘移出时，光线立即重置到中心位置，会看到一帧「闪回」效果。
  - 修复：在 `mouseleave` 时增加延迟定时器，先淡出光线，动画结束后再将 `--mouse-x/--mouse-y` 重置为 50%，避免突兀闪动。

- **主题色拖动圆点与进度条不对齐**
  - 现象：色相滑杆的圆形 thumb 略微偏离轨道中线，视觉上“压”在进度条上方。
  - 修复：增大轨道高度（6px → 8px），并将 thumb 的 `margin-top` 从 -6px 调整为 -5px，使其垂直居中。
